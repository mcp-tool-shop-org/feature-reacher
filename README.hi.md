**उन विशेषताओं का उपयोग करने से पहले ही उनका मूल्यांकन कर लें, ताकि वे तकनीकी ऋण न बन जाएं।**

फ़ीचर-रीचर आपके उत्पाद से संबंधित जानकारी (रिलीज़ नोट्स, दस्तावेज़, अक्सर पूछे जाने वाले प्रश्न) का विश्लेषण करता है और एक "**एडॉप्शन रिस्क ऑडिट**" तैयार करता है—यह विशेषताओं की एक क्रमबद्ध सूची है, जिसके साथ प्रमाण भी दिए गए हैं, जो उपयोगकर्ताओं द्वारा शायद कभी भी उपयोग न की जाने वाली सुविधाओं को दर्शाती है।

---

## यह क्या है

एक ऐसा उपकरण जो:
- उत्पाद दस्तावेज़ और रिलीज़ नोट्स को संसाधित करता है।
- प्रमाण के साथ विशेषताओं का उल्लेख निकालता है।
- विशेषताओं को उनकी नवीनता, दृश्यता और दस्तावेज़ घनत्व के आधार पर स्कोर देता है।
- कार्रवाई योग्य जोखिम निदान प्रदान करता है।

## यह क्या नहीं है

- कोई एनालिटिक्स प्लेटफॉर्म (उपयोग डेटा का कोई संग्रह नहीं)।
- कोई फ़ीचर फ़्लैग सिस्टम।
- आपके कोडबेस से जुड़ा कोई डैशबोर्ड।
- कोई ऐसा "मैजिक एआई" जो यह अनुमान लगाए कि उपयोगकर्ताओं को क्या चाहिए।

यह "**व्याख्या योग्य बुद्धिमत्ता**" है—प्रत्येक निदान के साथ उद्धृत प्रमाण और स्पष्ट तर्क दिए गए हैं।

---

## पहला चरण क्या प्रदान करता है

- **आर्टिफैक्ट अपलोड**: टेक्स्ट पेस्ट करें या .txt/.md फ़ाइलें अपलोड करें।
- **फ़ीचर निष्कर्षण**: हेडिंग, बुलेट लिस्ट, दोहराए गए वाक्यांश।
- **स्कोरिंग हेयूरिस्टिक्स**: नवीनता में कमी, दृश्यता संकेत, दस्तावेज़ घनत्व।
- **निदान इंजन**: 6 प्रकार के निदान, जिनमें ट्रिगरिंग संकेत और प्रमाण शामिल हैं।
- **क्रमबद्ध ऑडिट**: उपयोग जोखिम के आधार पर क्रमबद्ध विशेषताएं।
- **कार्रवाई अनुशंसाएँ**: प्रत्येक निदान के लिए कॉपी करने योग्य कार्रवाई।
- **निर्यात**: सादा पाठ और मुद्रण योग्य HTML रिपोर्ट।

## दूसरा चरण क्या जोड़ता है (दोहराव और प्रतिधारण)

- **ऑडिट इतिहास**: सभी ऑडिट IndexedDB में सहेजे जाते हैं, जिन्हें ब्राउज़, नाम बदलने और हटाने की सुविधा है।
- **ऑटो-सेव टॉगल**: ऑडिट एक बार चलाएं, स्वचालित रूप से (या मैन्युअल रूप से) सहेजें।
- **आर्टिफैक्ट सेट**: दोहराए जाने वाले ऑडिट वर्कफ़्लो के लिए नामित संग्रह।
- **ऑडिट तुलना**: साइड-बाय-साइड अंतर जो नए/हल किए गए जोखिमों और निदान परिवर्तनों को दिखाता है।
- **फ़ीचर ट्रेंड**: समय के साथ जोखिम प्रक्षेपवक्र का स्पार्कलिन विज़ुअलाइज़ेशन।
- **कार्यकारी सारांश**: पार्टनर के साथ साझा करने के लिए टेम्पलेट-आधारित सारांश (कोई एआई नहीं)।
- **टेस्ट सूट**: Jest + ts-jest जिसमें गार्डरेल टेस्ट शामिल हैं।

## तीसरा चरण क्या जोड़ता है (बाजार के लिए तैयार)

- **पब्लिक लैंडिंग पेज**: `/landing` जिसमें मूल्य प्रस्ताव और कॉल टू एक्शन (CTA) शामिल हैं।
- **डेमो मोड**: `/demo` जो तुरंत अनुभव के लिए नमूना डेटा स्वचालित रूप से लोड करता है।
- **कानूनी पृष्ठ**: गोपनीयता नीति, सेवा की शर्तें, सहायता संपर्क।
- **डेटा हैंडलिंग पैनल**: स्थानीय स्टोरेज के बारे में पारदर्शिता, जिसमें हटाने का विकल्प भी है।
- **ऑनबोर्डिंग टूर**: नए उपयोगकर्ताओं के लिए 4-चरणीय निर्देशित दौरा।
- **विधि पृष्ठ**: `/methodology` जो स्कोरिंग की व्याख्या करता है (कोई एआई प्रचार नहीं)।
- **त्रुटि सीमाएँ**: क्रैश-सेफ यूएक्स जिसमें नैदानिक निर्यात शामिल है।
- **ए11y उपयोगिताएँ**: कीबोर्ड नेविगेशन, ARIA हेल्पर, स्क्रीन रीडर समर्थन।
- **बाजार संपत्ति**: लिस्टिंग कॉपी, PDF दस्तावेज़, सबमिशन चेकलिस्ट।

## यह उपकरण जानबूझकर क्या नहीं करता है

- एनालिटिक्स प्लेटफॉर्म से कनेक्ट करना।
- GitHub, Jira या अन्य उपकरणों के साथ एकीकृत होना।
- फ़ीचर डिटेक्शन के लिए एमएल मॉडल का उपयोग करना।
- वास्तविक समय की निगरानी प्रदान करना।
- प्रमाणीकरण या खातों की आवश्यकता होना।
- कथा पीढ़ी के लिए एआई का उपयोग करना (केवल नियतात्मक टेम्पलेट)।

यह जानबूझकर किया गया है। तीसरा चरण एकीकरण जोड़ने से पहले बाजार की तत्परता को साबित करता है।

## शुरुआत कैसे करें

```bash
npm install
npm run dev
```

[http://localhost:3000](http://localhost:3000) खोलें।

### त्वरित परीक्षण

1. कुछ रिलीज़ नोट्स या दस्तावेज़ पेस्ट करें।
2. "ऑडिट चलाएं" पर क्लिक करें।
3. क्रमबद्ध विशेषताओं की सूची की समीक्षा करें।
4. प्रमाण और अनुशंसाएँ देखने के लिए विशेषताओं का विस्तार करें।
5. रिपोर्ट निर्यात करें।

### दूसरा चरण वर्कफ़्लो

**दोहराव प्रवाह:**
1. ऑडिट चलाएं → इतिहास में स्वचालित रूप से सहेजा गया।
2. सहेजे गए ऑडिट को ब्राउज़ करने के लिए `/history` पर जाएं।
3. दोहराए जाने वाले रनों के लिए आर्टिफैक्ट संग्रह सहेजें।
4. `/compare` पर दो ऑडिट की तुलना करें।
5. `/trends` पर रुझान देखें।

**पार्टनर शेयरिंग प्रवाह:**
1. ऑडिट चलाएं → "निर्यात" पर क्लिक करें → "कार्यकारी सारांश"।
2. या दो ऑडिट की तुलना करें → "तुलना रिपोर्ट निर्यात करें"।

## परियोजना संरचना

```
/src/app       # Next.js app router pages
/src/domain    # Core feature model and business logic
/src/analysis  # Diagnostics, heuristics, scoring, diff, trend
/src/storage   # IndexedDB persistence layer
/src/ui        # Reusable UI components
/tests         # Jest test suite
/docs          # Project documentation
```

### मुख्य फाइलें

- `src/domain/feature.ts` - मानक फ़ीचर मॉडल
- `src/domain/diagnosis.ts` - निदान के प्रकार और गंभीरता
- `src/analysis/extractor.ts` - फ़ीचर निष्कर्षण के लिए दिशानिर्देश
- `src/analysis/scoring.ts` - हालिया/दृश्यता स्कोरिंग
- `src/analysis/diagnose.ts` - निदान इंजन
- `src/analysis/ranking.ts` - जोखिम रैंकिंग और ऑडिट निर्माण
- `src/analysis/actions.ts` - कार्रवाई की सिफारिशें
- `src/analysis/export.ts` - रिपोर्ट निर्माण + कार्यकारी सारांश
- `src/analysis/diff.ts` - ऑडिट तुलना इंजन
- `src/analysis/trend.ts` - फ़ीचर ट्रेंड विश्लेषण
- `src/storage/types.ts` - डेटा भंडारण के प्रकार
- `src/storage/indexeddb.ts` - IndexedDB कार्यान्वयन

### डिजाइन सिद्धांत

1. **कोई रहस्य नहीं**: प्रत्येक निदान को उद्धृत प्रमाणों के साथ समझाया जा सकता है।
2. **पहले दिशानिर्देश**: मशीन लर्निंग का उपयोग केवल तभी करें जब दिशानिर्देश अपर्याप्त साबित हों।
3. **निश्चित**: समान इनपुट हमेशा समान आउटपुट उत्पन्न करता है।
4. **पारदर्शी**: उपयोगकर्ता किसी भी निष्कर्ष को उसके स्रोत तक ट्रैक कर सकते हैं।
